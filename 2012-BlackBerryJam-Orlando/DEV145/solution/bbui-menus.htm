bbUI.js Menus

The bbUI toolkit builds the application's UI in the most optimized fashion for the target operating system. It follows a methodology of a single web page that has screens loaded into it as HTML fragments: each screen is its own HTML fragment file.

The toolkit then uses AJAX to **push** and **pop** screens off of the stack. The toolkilt manages the screen stack and loading the content. This ensures the best use of device memory.

You can also define screen loading transition effects. Currently the fade effect is supported.

Tasks:
  1. Edit bbui-screens.htm
  2. Add onclick="bb.pushScreen('bbui-newscreen.htm', 'newscreen');" to the <button>
  3. Test using Ripple, click on the "open new screen" button and verify that "New Screen" is opened
  4. Click the back button. Verify "New Screen" is popped off display stack and return to bbui-screens.htm.
  5. Edit bbui-newscreen.htm
  6. Add data-bb-effect="fade" to the <div data-bb-type="screen"> element
  7. Test using Ripple, click on the "open new screen" button and verify that "New Screen" is opened with a fade effect.

Tips:

  • bb.popScreen() will always remove the last screen added to the display stack.
